# 配置管理规范
## 1. 配置库结构与权限规则
**说明**：
1. 下表前5列描述配置库结构；后6列描述项目各角色对配置库的目录项的访问权限。
2. 权限具有继承性，即如果子目录的权限没有重新定义，则会继承父目录的权限。
3. 权限标记：
   - R ：读权限
   - W ：写权限
   - N ：无权限
4. 其它标记：
   - <yymmdd> ：日期简写。
   - <评审对象>：接受评审的文档的名称（不含产品简称或项目简称前缀，也不含文档扩展名）。例如：软件需求规格说明书。
   - <版本号>：文档/代码版本号。例如：20260611-1。
